The chip statistics characterize the performed hybridization in terms of chip-related parameters.

Sensitvity Profiles

The Sensitvity Profiles reflect the relative binding strength of each base at each position in the probe sequence. This behaviour is different for PM and MM sequences as well as for specific and non-specific hybridization.

Parameter settings

Analyzed chip: intensity-file
Chip-type: chip-file
Configuration file: config-file
Date of analysis: startingTime
Method for averaging: averaging-method
Method to robustly determine probeset average intensity (mean, median or tukey)
Background substraction divides the chip in x² areas: x = background-subtraction-zones
Multiplicative factor applied to the background value to be substracted from each intensity:   background-subtraction-ratio
Maximum number of probes used to compute the sequence profiles (0 = use all probes): probeset-limit
Rank of the sequence profile model (1=mononucleotid, 2=dinucleotid): sequence-profile-rank
Detect-ip: detect-ip
  Method to detect inflection point: 0=line and parabola; 1=two straight lines in convenient intervals;
  2=first deviation; 3=maximum of second deviation
Digitize-intervals: digitize-intervals
  Number of intervals the hookcurve is digitized to fit the functions to detect inflection point.
  (0 equals no digitization)
Expression-measures: expression-measures
  What expression measures shall be calculated (log and glog-scale)
gLog-Parameter: gLog-parameter
Interval-count: interval-count
  Specifies the number of intervals for the FitTwoStraightLines algorithm to detect the inflection point.
Maskfile: maskfile
Filter of probeset-size: probeset-size-filter
Saturation-computation: saturation-computation
Set kink point of uncorrected hookcurve (0 to compute): set-uncorrected-kinkpoint
Set kink point of corrected hookcurve (0 to compute): set-corrected-kinkpoint
  You can adjust the kink point if the calculated does not fit.
(tiling only)
Probability above which a probe is considered target-present: shuffling-specific-treshold
Number of probes that a probeset optimally contains: shuffling-optimal-probes-per-set
Number of probes that a probeset maximally contains: shuffling-maximal-probes-per-set
Maximum allowed distance between adjacent probes within a probeset: shuffling-max-probe-distance
Size of moving average window during shuffling: shuffling-window-size
